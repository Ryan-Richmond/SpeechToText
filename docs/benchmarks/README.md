# Vox Benchmarks

Benchmark results are stored here as markdown reports. Each run produces one file:

```
docs/benchmarks/
├── README.md              (this file)
├── prompt-eval-YYYYMMDD.md    Generated by vox-prompt-eval
└── model-bench-YYYYMMDD.md   Generated by vox-model-bench
```

## prompt-eval reports

Produced by `vox-prompt-eval` (`scripts/score.py`). Format:

```markdown
# Prompt Eval — 2026-04-14
## Summary
- Overall: 38/40 (95%)
- Threshold: 85%
- Result: PASS
## By Category
| Category | Passed | Total | Rate |
...
```

Commit every report. This is our quality history.

## model-bench reports

Produced by `vox-model-bench`. Each report covers one device/model combination:

```markdown
# Model Bench — 2026-04-14 — M1 Mac 16GB
## Models
- Whisper medium.en (Core ML)
- Gemma 4 E4B Q4_K_M

## Results (p50 over 20 utterances)
| Metric     | Value      | Target     | Pass? |
|------------|------------|------------|-------|
| WER        | 3.8%       | ≤ 5.0%     | ✓     |
| Latency    | 1.7s       | ≤ 2.0s     | ✓     |
| Peak RAM   | 4.3 GB     | ≤ 5.0 GB   | ✓     |
| Battery    | 2.1%/hr    | ≤ 5%/hr    | ✓     |
```

Run `vox-model-bench` on real hardware after every model change and before every release.
